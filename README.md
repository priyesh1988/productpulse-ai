# ProductPulse AI

**AI-native product operating system for modern teams**

ProductPulse AI is a product-focused, production-ready starter repository for teams that want to turn customer signals, telemetry, tickets, and product usage into **prioritized product decisions** using **LLMs, vector search, and workflow automation**.

It is designed as a modern MVP product you can extend into a real SaaS product.

## What it does

- Ingests product feedback, incidents, churn signals, feature requests, and usage events
- Uses LLMs to summarize, cluster, and prioritize product opportunities
- Stores semantic context in a vector layer for grounded retrieval
- Produces recommendations such as:
  - what to build next
  - what customers are complaining about most
  - what feature adoption is slipping
  - what release risk is increasing
- Exposes APIs for product intelligence workflows
- Ships with a dashboard UI, API service, local dev setup, CI, and Kubernetes manifests

## Why this repo is product-focused

Most AI repos stop at “chat with your docs.”

This one is centered on **product evolution**:

- feedback-to-roadmap intelligence
- issue-to-opportunity triage
- AI-assisted release and adoption insights
- customer voice + telemetry + ticket unification
- product ops workflows with human review

## Local setup

### 1. Clone
```bash
git clone <your-repo-url>
cd productpulse-ai
```

### 2. Configure environment
```bash
cp backend/.env.example backend/.env
cp frontend/.env.local.example frontend/.env.local
```

Set your keys in `backend/.env`:

```env
OPENAI_API_KEY=your_key_here
OPENAI_MODEL=gpt-5.4
POSTGRES_URL=postgresql://postgres:postgres@localhost:5432/productpulse
QDRANT_URL=http://localhost:6333
REDIS_URL=redis://localhost:6379/0
```

### 3. Start dependencies
```bash
docker compose up -d postgres qdrant redis
```

### 4. Start backend
```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

### 5. Start frontend
```bash
cd frontend
pnpm install
pnpm dev
```

## Product differentiators

### 1. Not just a chatbot
This repo is designed around product operating workflows, not generic Q&A.

### 2. Multi-signal intelligence
It combines customer voice, usage data, and release context.

### 3. LLM with retrieval foundation
It includes semantic context hooks instead of relying only on prompts.

It has service boundaries, configs, tests, containers, CI, and Kubernetes manifests.

### 5. Extensible for real teams
You can plug in Jira, Linear, HubSpot, Zendesk, Segment, or Snowflake later.

## Failure scenarios handled in the design

- Missing LLM key → API returns clear configuration error
- Empty signal batch → validation rejects bad requests
- Vector store unavailable → service can degrade to direct scoring path
- Model timeout → API surface returns safe fallback message
- Over-prioritization of noisy feedback → scoring kept transparent and auditable
